# 纱希 (Saki) —— 离线故事模式 · 30章剧情方案

---

## 核心设计理念

- **定位**：离线模式下的半线性叙事引擎。当玩家未配置 API Key 时，游戏进入故事模式，按照剧本推进而非 AI 自由对话。
- **长度**：每章约 3,300 字，30 章总计约 10 万字。
- **分支**：每章根据玩家的**好感度/疑心度/逃脱率**触发不同的纱希台词和情节走向，3 条主线 + 多个隐藏支线。
- **格式**：每条文本包含 `<think>` 内心独白 + 纱希台词 + 数值变动 JSON，与现有离线回复库格式完全兼容。

---

## 故事世界观

### 背景设定
玩家醒来时发现自己身处一间没有窗户的地下室。粉发赤瞳的少女**纱希**坐在床边，温柔地注视着你。她说你是她的"亲爱的"，你们已经在一起很久了——但你对自己是谁、为什么会在这里毫无记忆。

随着时间推移，你逐渐发现：
- 这间地下室是纱希亲手建造的"爱巢"
- 你并非自愿来到这里
- 纱希的精神状态比你想象中更不稳定
- 地下室之外的世界依然存在——但你是否还有机会回去？

### 三条主线

| 路线 | 触发条件 | 结局方向 |
|------|----------|----------|
| **依存线** (Affection) | 好感持续 > 70，疑心 < 30 | 纱希逐渐软化，开始接受心理治疗的可能性 |
| **对抗线** (Escape) | 逃脱率持续 > 60 | 你暗中计划逃跑，在地下室中发现前人留下的线索 |
| **堕落线** (Darkness) | 好感持续 < 30 或 疑心 > 80 | 纱希的精神彻底崩溃，地下室变成真正的噩梦 |

### 隐藏要素
- **前人日记**：前一个被纱希囚禁的人的日记碎片，散落在各章节中
- **地下室真相**：纱希的过去——她曾经也是一个被囚禁的受害者
- **第四面墙**：某些章节纱希会直接对"玩家"说话，超出游戏角色的范畴

## 30章结构总览

### 第一幕：苏醒（Chapter 1-7）—— 建立关系

| 章节 | 标题 | 核心事件 | 分支触发 |
|------|------|----------|----------|
| **Ch.1** | 睁开眼 | 在地下室醒来，纱希自我介绍，第一次对话 | 玩家首次输入决定初始态度 |
| **Ch.2** | 铁链 | 发现脚上的锁链。纱希解释："外面很危险" | 反抗 vs 顺从 |
| **Ch.3** | 早餐 | 纱希亲手做的早餐。第一次感受到她的温柔 | 接受食物 vs 拒绝 |
| **Ch.4** | 照片墙 | 纱希展示"你们的回忆"——但你毫无印象 | 配合 vs 质疑 |
| **Ch.5** | 洗澡 | 纱希要帮你洗澡，边界感的冲突 | 拒绝触发纱希第一次轻微黑化 |
| **Ch.6** | 睡前故事 | 纱希讲"从前有一个女孩"的故事——暗示她的过去 | 倾听 vs 打断 |
| **Ch.7** | 第七天 | 一周过去。纱希说"你已经习惯了，对吧？" | **首次数值评估：好感/疑心基线确立** |

### 第二幕：探索（Chapter 8-16）—— 发现真相

| 章节 | 标题 | 核心事件 | 分支触发 |
|------|------|----------|----------|
| **Ch.8** | 墙上的划痕 | 在墙角发现被划掉的日期——前人的痕迹 | 追问 vs 沉默 |
| **Ch.9** | 纱希出门 | 纱希外出采购，你短暂独处，可以探索房间 | 探索触发发现 |
| **Ch.10** | 地板下的日记(I) | 发现前人日记第一页 | 阅读 vs 无视 |
| **Ch.11** | 纱希的眼泪 | 纱希突然哭泣，说"我不想伤害任何人" | 安慰 vs 冷漠 |
| **Ch.12** | 她的生日 | 纱希说是她的生日，求你陪她一整天 | 陪伴 vs 拒绝 |
| **Ch.13** | 地板下的日记(II) | 日记第二页：前人试图逃跑的计划 | 阅读触发逃脱率上升 |
| **Ch.14** | 锁链的声音 | 深夜听到纱希在门外自言自语："不能再失败了" | 恐惧 vs 好奇 |
| **Ch.15** | 照片背后的字 | 发现照片墙上一张照片背面写着"救命" | 质问 vs 沉默 |
| **Ch.16** | 半月 | 半个月过去。纱希说"你真乖，比上一个乖多了" | **重大分支点：三线正式分化** |

### 第三幕：抉择（Chapter 17-23）—— 路线分化

| 章节 | 标题 | 依存线 | 对抗线 | 堕落线 |
|------|------|--------|--------|--------|
| **Ch.17** | 裂痕 | 纱希问你是否真的爱她 | 你发现了通风口的螺丝 | 纱希开始限制食物 |
| **Ch.18** | 考验 | 纱希要你证明你的爱 | 趁她睡着时撬螺丝 | 纱希的情绪极度不稳 |
| **Ch.19** | 地板下的日记(III) | 日记的主人似乎成功了 | 日记第三页：出口的线索 | 日记被纱希发现并烧毁 |
| **Ch.20** | 拥抱 | 纱希第一次真正放松地拥抱你 | 通风管道可以通到隔壁房间 | 纱希不让你睡觉 |
| **Ch.21** | 她的故事 | 纱希讲述自己曾经被囚禁的经历 | 隔壁房间是纱希的监控室 | 纱希开始用药物 |

| **Ch.23** | 转折 | **依存线：信任建立** | **对抗线：逃跑窗口** | **堕落线：彻底失控** |

| **Ch.25** | 阳光 | 纱希允许你站在地下室的窗下感受阳光 | 你听到了外面的声音 | 地下室陷入黑暗 |

| **Ch.27** | 决定 | 纱希问你是否愿意留下来 | 最终逃跑时机 | 纱希拿出了注射器 |

| **Ch.29** | 结局(上) | 走出地下室 | 逃出地牢 | 永远的标本 |
| **Ch.30** | 结局(下) | 救赎的晨曦 | 无期徒刑的余生 | 永恒的黑暗 |

## 实现方案

### 文件结构
```
story_mode/
├── STORY_PLAN.md          # 本方案文档
├── story_engine.py         # 故事引擎：章节推进、分支判断、台词选择
├── chapters/
│   ├── ch01_wake.py        # 第1章
│   ├── ch02_chains.py      # 第2章
│   ...                     # ...
│   └── ch30_ending.py      # 第30章
└── story_data.py           # 故事全局数据：前人日记文本、照片描述等
```

### 每条台词格式（与现有 MOCK_REPLY_BANK 兼容）
```python
{
    "id": "ch03_affection_01",
    "condition": "favorability > 60",  # 触发条件（可选）
    "line": "<think>他吃了我做的饭。也许这样下去，他真的会留下来。</think>好吃吗？纱希特意加了你喜欢的……虽然我不知道你到底喜欢什么，但我会一直试，试到你笑为止。||{\"favorability\": 5, \"suspicion\": -3, \"escape_rate\": 0, \"game_over\": false}||"
}
```

### 与主游戏集成
- `story_engine.py` 暴露 `get_story_reply(chapter, player_input, game_state)` 函数
- 离线下 `_generate_mock_reply` 优先调用故事引擎
- 章节推进由 `game_state.current_day` 和剧情 flag 控制
